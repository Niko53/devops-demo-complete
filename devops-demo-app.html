Overview

Che cosa puoi mostrare durante una lezione DevOps.

La demo è costruita per essere spiegata a livelli: prima il frontend statico, poi il container, quindi la pipeline, il deploy e infine l’osservabilità. In questo modo gli studenti vedono subito un caso d’uso coerente e riutilizzabile.

Frontend semplice

Una singola pagina HTML, CSS e JavaScript: perfetta per introdurre Dockerfile, image build, versioning e deploy senza aggiungere complessità inutile.

Endpoint tipici

Health, readiness, metrics, version e config sono i classici endpoint o artefatti che si usano come esempio per reverse proxy, probe Kubernetes e monitoring.

Scenario reale

Puoi usarla per simulare rollout, variabili d’ambiente, incidenti controllati, feature flag e differenze tra ambienti dev, staging e production.

Esempio pipeline

Una sequenza molto chiara da usare per spiegare la CI/CD.

1. Build
Checkout del repository, validazione del contenuto e creazione dell’immagine container.
2. Test
Linting, smoke test e verifica dei file pubblicati.
3. Publish
Push dell’immagine su un registry come GHCR o Docker Hub.
4. Deploy
Aggiornamento di un Deployment Kubernetes con nuova immagine e rollout controllato.
name: deploy-demo-app
on:
  push:
    branches: [main]
jobs:
  build-and-deploy:
    steps:
      - uses: actions/checkout@v4
      - run: docker build -t ghcr.io/org/devops-demo:${GITHUB_SHA} .
      - run: docker push ghcr.io/org/devops-demo:${GITHUB_SHA}
      - run: kubectl set image deployment/devops-demo app=ghcr.io/org/devops-demo:${GITHUB_SHA}

Idee per le lezioni

Come usare questa app durante il corso.

  • Mostrare differenza tra build locale e build container.
  • Associare una release a una versione leggibile in UI.
  • Far vedere come cambiano le metriche dopo un deploy.
  • Spiegare readiness e liveness probe con esempi pratici.
  • Simulare un rollback dopo un incremento dell’error rate.

FAQ

Domande tipiche da affrontare in aula.

Queste risposte possono essere utili anche come traccia per il docente, per contestualizzare la demo durante le lezioni su Docker, CI/CD, Kubernetes e monitoring.

Perché usare un’app così semplice?

Perché consente di spiegare bene il flusso DevOps senza sprecare tempo sul dominio applicativo. L’attenzione resta su build, deploy, configurazione, observability e troubleshooting.

È adatta a Docker e Kubernetes?

Sì. Proprio perché è piccola, è ideale per mostrare containerizzazione, image tagging, rollout e probe applicativi con pochi passaggi chiari.

Posso aggiungere un backend reale in seguito?

Certo. Questa base può evolvere facilmente in una piccola app Node, Python o .NET con endpoint veri, mantenendo la stessa struttura visiva e didattica.
